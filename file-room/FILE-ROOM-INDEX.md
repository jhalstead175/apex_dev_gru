## üè∑Ô∏è NAMING CONVENTIONS

### File Naming Standard
```
[CATEGORY]-[DESCRIPTION]-[TYPE]-v[VERSION].xlsx
```

**Examples:**
- `UNDERWRITING-Single-Family-BRRRR-TEMPLATE-v2.1.xlsx`
- `CONSTRUCTION-Residential-Rehab-BUDGET-v1.5.xlsx`
- `TAX-Cost-Segregation-ANALYZER-v3.0.xlsx`
- `LOAN-DSCR-Commercial-CALCULATOR-v1.2.xlsx`

### Version Numbering
- **Major Version (X.0):** Significant redesign or functionality change
- **Minor Version (1.X):** New features, additional sections
- **Patch (1.1.X):** Bug fixes, formula corrections (optional third digit)

### File Types
- **TEMPLATE:** Blank model ready for new deal input
- **ANALYZER:** Tool for quick calculations
- **CALCULATOR:** Single-purpose computational tool
- **MODEL:** Comprehensive financial model
- **TRACKER:** Data logging and monitoring tool
- **DASHBOARD:** Visualization and reporting tool

### Font Standards
- **Headers:** Calibri Bold 11pt or 12pt
- **Body Text:** Calibri Regular 10pt or 11pt
- **Numbers:** Calibri Regular 10pt, right-aligned
- **Currency:** Accounting format with $ and (parentheses) for negatives

### Cell Protection
- **Locked:** All cells EXCEPT input cells
- **Hidden Formulas:** All calculation cells should hide formulas from end users
- **Sheet Protection:** Enable with password for production models

## üìä STANDARD MODEL STRUCTURE

Every Excel model should follow this structure:

### Tab Organization
1. **COVER** - Model title, version, instructions, table of contents
2. **INPUTS** - All user-input variables (light blue cells)
3. **ASSUMPTIONS** - Fixed assumptions, market data (gray cells)
4. **CALCULATIONS** - All computational logic (hidden or light green)
5. **OUTPUTS** - Summary results, dashboards (light yellow)
6. **CHARTS** - Visualizations and graphs
7. **SUPPORTING** - Amortization tables, detailed schedules
8. **DATA** - Raw data, lookups, validation lists
9. **CHANGELOG** - Version history and modification log

### Mandatory Sections (All Models)
- **Model Information Block:**
  - Model Name
  - Version Number
  - Date Created / Last Modified
  - Author
  - Description
  - Revision History

- **Input Validation:**
  - Drop-down lists where applicable
  - Data validation rules
  - Error checking formulas
  - Input range checks

- **Output Summary:**
  - Key metrics dashboard
  - Go/No-Go decision indicators
  - Sensitivity table (where applicable)

## üöÄ USAGE GUIDELINES

### For New Users
1. Review this index to understand file organization
2. Start with templates in `10-REFERENCE-DOCS/Templates`
3. Review `APEX-STANDARDS-Modeling-v1.0.md` for formatting rules
4. Use `FORMULA-LIBRARY-v1.0.xlsx` as formula reference
5. Complete Excel training modules in `10-REFERENCE-DOCS/Training`

### For Model Developers
1. Always start from existing template when creating new models
2. Follow color coding and formatting standards religiously
3. Document all assumptions clearly
4. Include error checking and input validation
5. Test with extreme values (stress test)
6. Peer review before releasing to production
7. Maintain version history in CHANGELOG tab

### For Deal Teams
1. Copy template from File Room to deal folder
2. Rename file to include deal address/name
3. Never modify File Room master templates directly
4. Save deal files with descriptive names
5. Archive completed deals appropriately
